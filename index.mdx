## Why Pinia Colada?

Instead of writing this every time you fetch data:

```vue
<script setup lang="ts">
import { shallowRef, ref } from 'vue'
import { fetchUsers } from '../api/users'

const isLoading = ref(false)
const isPending = ref(true)
const error = shallowRef(null)
const data = shallowRef()

async function refresh() {
  isLoading.value = true
  try {
    data.value = await fetchUsers()
    error.value = null
    isPending.value = false
  } catch (err) {
    error.value = err
  } finally {
    isLoading.value = false
  }
}

refresh()
</script>
```

You write this:

```vue
<script setup lang="ts">
import { fetchUsers } from '../api/users'
import { useQuery } from '@pinia/colada'

const { data, error, isPending, isLoading, refresh } = useQuery({
  key: ['users'],
  query: fetchUsers,
})
</script>
```

And get automatic caching, request deduplication, background refetching, and more — for free.

## Key features

<CardGroup cols={2}>
  <Card title="Automatic caching" icon="database">
    Smart client-side caching with stale-while-revalidate and request deduplication across components.
  </Card>
  <Card title="Query invalidation" icon="arrows-rotate">
    Invalidate and refetch queries automatically when mutations succeed.
  </Card>
  <Card title="Optimistic updates" icon="bolt">
    Update your UI instantly before the server responds, with automatic rollback on failure.
  </Card>
  <Card title="Infinite queries" icon="infinity">
    Built-in support for paginated and infinite scroll data patterns.
  </Card>
  <Card title="SSR support" icon="server">
    Automatic server-side rendering via `onServerPrefetch` — no `await` required.
  </Card>
  <Card title="Plugin system" icon="puzzle-piece">
    Extend Pinia Colada with official and community plugins: retry, auto-refetch, delay, and more.
  </Card>
  <Card title="TypeScript first" icon="code">
    Full type safety throughout — type-safe cache access, query keys, and mutation parameters.
  </Card>
  <Card title="Tiny bundle" icon="feather">
    ~2kb baseline, zero dependencies beyond Pinia and Vue, fully tree-shakeable.
  </Card>
</CardGroup>
